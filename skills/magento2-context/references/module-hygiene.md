# Module Hygiene (shared contract)

Single source of truth for the **packaging/compliance baseline** every Magento 2 module must meet so
that code generated by any `magento2-*` skill passes `magento2-marketplace-prep` and Adobe Marketplace /
EQP on the first try. The audit contract lives in
`magento2-marketplace-prep/references/eqp-checklist.md`; this file is the **generator-side** mirror —
what every skill that writes files into a module must emit.

Every generator skill that creates `.php` files or touches a module's `composer.json` MUST apply the
rules below. Audit/review skills (which never write into a module) are exempt.

---

## 1. Copyright header on every PHP file

Every `.php` file generated into a module (production **and** test) carries this header, inserted
immediately after the `<?php` line:

```php
<?php
/**
 * Copyright © {Vendor}. All rights reserved.
 * See LICENSE.txt for license details.
 */
declare(strict_types=1);
```

- The header is **generic on purpose** — it points at `LICENSE.txt` rather than restating license
  terms, so the same block is correct for both a proprietary EULA and an OSI license.
- **Do not hand-write it per file.** After generating files, run the shared stamper:

  ```bash
  ${CLAUDE_PLUGIN_ROOT}/skills/magento2-context/scripts/add-license-headers.sh {module_path} {Vendor}
  ```

  It stamps every `.php` under `{module_path}`, is **idempotent** (skips files that already carry the
  header — safe to re-run / safe when adapting an existing module), and skips files whose first line is
  not `<?php`. If the script is unavailable, prepend the block above by hand to each new `.php` file.

## 2. LICENSE file

A module that declares a `license` in `composer.json` MUST ship a matching license **file**:

- `LICENSE.txt` at the module root, contents matching the `license` field.
- For `"proprietary"`, use the proprietary EULA (see
  `magento2-module-create/templates/LICENSE.txt`). For an SPDX id (`OSL-3.0`, `AFL-3.0`, `MIT`, …), use
  that license's standard text. Keep file and field in sync.

## 3. composer.json metadata

- **`authors`** — non-empty array of `{name, email}`. Derive `name` from `git config user.name`
  (fallback `gh api user` for the GitHub identity) and `email` from `git config user.email`. **Never**
  use `{Vendor}` as the author name. Ask the user only if both git and GitHub identities are empty.
- **Bounded version constraints — never `"*"`.** EQP rejects wildcard / `dev-` / `@dev` constraints.
  When adding a `require` entry, resolve a bounded constraint instead of guessing:

  ```bash
  ${CLAUDE_PLUGIN_ROOT}/skills/magento2-context/scripts/resolve-dep-constraint.sh <vendor/package> [project_root]
  ```

  It resolves the installed version (via `composer show`, then `composer.lock`, then `src/composer.json`)
  and prints a bounded `>=X.Y.0` floor. Mirror the operator style already used by the store's
  `src/composer.json`. If nothing resolves, ask the user — do not fall back to `"*"`.

## 4. `.gitignore`

A module should ship a `.gitignore` (excludes `vendor/`, `node_modules/`, `*.log`, `.DS_Store`). See
`magento2-module-create/templates/gitignore`.

---

## Enforcement

- `magento2-module-create/scripts/verify-created.sh` gates a freshly created module on all of the above
  (LICENSE present, `.gitignore`, `authors`, no `*` constraints, header on every `.php`).
- `tests/test-license-header-coverage.sh` (skill-pack contract test) fails if any skill that ships
  `templates/*.php` does not wire in the shared stamper.
- The deep audit is `magento2-marketplace-prep` — run it for the full readiness verdict.
